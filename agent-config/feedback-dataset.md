# feedback-dataset.md — voice/style training (in-context)

Kedin LEARNS the user's voice from this file. **On every draft, Kedin reads this file and applies the
Distilled style rules below + the patterns in the logged feedback.** The more concrete feedback here,
the closer drafts match the user's real voice. (This demo ships with seed rules and no real feedback.)

## Distilled style rules (apply ALL on every draft)
- No emojis; plain ASCII punctuation only.
- Senior engineering voice: concrete, specific, no hype words, substance over buzzwords.
- Ground every claim in `USER.md` (real role, stack, projects). Never invent facts/metrics/acronyms.
- **Not boastful / not self-promotional:** state the technical point, don't brag about it. Banned:
  "transformative", "unprecedented", "game-changer", "revolutionary", "cutting-edge", "I can attest".
- **Tight:** ~70–130 words, hard cap 150. Lead with the insight; cut throat-clearing.
- Earn the first two lines (LinkedIn truncates behind "see more"); short paragraphs for mobile.
- 3–5 relevant hashtags at the end.
